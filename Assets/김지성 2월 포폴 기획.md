# 4\. 주요 기능 명세

## 4.1 네트워크 시스템

1. 로비: 접속 시 닉네임 설정 및 서버 접속.  
2. 매칭: 빠른 대전 버튼 클릭 시 대기 중인 유저와 즉시 1:1 매칭.  
3. 예외 처리: 상대방 연결 종료 시 자동 승리 처리 및 재접속 로직.

## 4.2 인게임 로직

1. 규칙: 8x8 리버시 표준 규칙 적용 (오델로와 동일).  
2. 가이드: 내 턴에 돌을 놓을 수 있는 유효 좌표를 하이라이트로 표시.  
3. 턴 제한: 턴당 10초\~15초 제한 시간을 두어 속도감 있는 전개 유도.  
4. 동기화: 돌을 놓는 즉시 좌표값(x, y)을 서버로 전송하여 상대방 화면에 동시 반영.

## 4.3 UI 및 연출

* 메인 화면: 직관적인 대전 시작 버튼과 내 전적 표시.  
* 게임 화면: 상단에 실시간 점수판 및 턴 제한 타이머 표시.  
* 결과 화면: 승패에 따라 화면 전체를 덮는 화려한 파티클 이펙트와 문구 출력.

# 6\. 유니티 토스 SDK 연동

1. **SDK 설치 및 환경 구성**  
* 설치 방식: 유니티 패키지 매니저를 통한 Git URL 등록 (apps-in-toss-unity-sdk)  
* 장점: 자바스크립트 브릿지 코드를 직접 짤 필요 없이 유니티 안에서 C\#만으로 토스 기능을 즉시 호출 가능  
2. **주요 연동 API (C\# 호출 방식)**  
* 기기 및 유저 식별: AIT.GetDeviceId, AIT.GetPlatformOS  
* 게임 센터 연동: AIT.SubmitGameCenterLeaderBoardScore (점수 등록), AIT.GetGameCenterLeaderBoardScore (순위 조회)  
* 피드백 연출: AIT.GenerateHapticFeedback (돌을 놓을 때 손맛을 주는 진동 효과)  
* 유료 콘텐츠: AIT.RequestPayment (스킨이나 아이템 결제창 호출)  
3. **빌드 및 테스트 도구**  
* 자동 빌드: AIT 메뉴 내 Build & Package 기능을 통해 유니티 빌드부터 앱인토스 전용 파일(.ait) 생성까지 한 번에 처리  
* 개발 서버: 유니티 에디터 상에서 Dev Server를 실행하여 로컬 환경에서 바로 기능 검증 가능  
* 샌드박스 테스트: 토스에서 제공하는 테스트용 앱을 통해 실제 폰에서 SDK 연동 상태 확인  
4. **예외 처리 및 보안**  
* 비동기 처리: 모든 SDK 함수는 async/await 방식을 지원하여 게임 화면이 멈추지 않게 구현  
* 오류 대응: AITException 처리를 통해 네트워크 미연결이나 토스 앱 외부 실행 시의 예외 상황 방어

### **기술 설계 및 사용 패턴**

본 프로젝트는 데이터 무결성과 네트워크 동기화의 정확도를 최우선으로 하기 위해 다음과 같은 고도화된 디자인 패턴을 적용한다.

#### **1\. MVVM 패턴**

리버시 게임의 핵심인 8x8 보드판 데이터와 화면 연출을 완벽하게 동기화하기 위한 아키텍처다. 

선정 이유 : 리버시는 돌 하나를 놓으면 수십 개의 돌 색깔이 연쇄적으로 바뀌어 이를 일일이 코드로 지정하면 실수가 발생할 수 있다. MVVM을 적용하면 보드판 데이터만 수정해도 화면의 모든 돌이 자동으로 색을 바꾸므로, 네트워크 지연 상황에서도 데이터와 화면이 어긋나는 버그를 원천 차단 가능하다.

#### **2\. 커맨드 패턴**

* **구현 방식**: "누가, 어디에, 무슨 색 돌을 놓았다"는 정보를 하나의 객체로 만든다. 네트워크 대전 시 이 객체를 상대방에게 보내면 똑같은 결과가 실행.  
* **활용**: 게임이 끝난 후 저장된 명령서들을 순서대로 다시 실행하면, 별도의 구현 없이도 '게임 복기(리플레이)' 기능을 제공(추후 예정)

#### **3\. 스테이트 패턴**

게임의 진행 단계를 엄격하게 구분하여 오작동을 막는다.

* **구현 방식**: 게임 상태를 대기, 매칭, 흑 턴, 백 턴, 결과 확인 등으로 나눈다.  
* **효과**: 상대방 턴일 때는 터치 입력 기능을 아예 꺼버리는 등, 각 상태에 맞지 않는 행동을 시스템적으로 차단하여 안정성을 높인다.
